# ブログ投稿手順

## 投稿ファイルの場所

```
src/content/blog/
```

ここにMarkdown（`.md`）ファイルを追加するだけで記事が増えます。

## ファイル名のルール

```
src/content/blog/記事のslug.md
```

- ファイル名がURLのslugになります
  - 例：`why-i-built-this-site.md` → `/blog/why-i-built-this-site`
- 英語・ハイフン区切りで付ける
- 短くて分かりやすい名前が理想

## frontmatter（必須）

ファイルの先頭に以下のYAMLブロックを必ず書いてください。

```yaml
---
title: "記事タイトル"
description: "記事の概要（1〜2文）。ブログ一覧ページと検索結果に表示されます。"
pubDate: YYYY-MM-DD
tags: ["タグ1", "タグ2"]
---
```

| フィールド | 必須 | 説明 |
|---|---|---|
| `title` | ✅ | 記事タイトル |
| `description` | ✅ | 一覧ページに表示される概要文 |
| `pubDate` | ✅ | 公開日（`2026-05-16` 形式） |
| `updatedDate` | ❌ | 更新日（省略可） |
| `tags` | ❌ | タグの配列（省略可） |

### タグの使い方

タグはブログ一覧のフィルターとRelation Graphに反映されます。  
同じトピックの記事には同じタグを付けると関連記事がグラフに表示されます。

現在使っているタグ例：
- `portfolio` — このサイト・ポートフォリオ関連
- `vibe-coding` — AI/Claude Codeを使った開発
- `design` — デザイン関連
- `dev` — 開発全般
- `career` — キャリア・仕事関連

## 記事の書き方

frontmatterの後ろに普通のMarkdownで書くだけです。

```markdown
---
title: "記事タイトル"
description: "概要"
pubDate: 2026-05-16
tags: ["dev"]
---

ここから本文。

## 見出し

本文テキスト。

## 別の見出し

続き。
```

使えるMarkdown記法：
- `**太字**`、`*斜体*`
- `## 見出し2`、`### 見出し3`
- `` `インラインコード` ``
- コードブロック（` ```js `〜` ``` `）
- `> 引用`
- `- リスト`
- `[リンクテキスト](URL)`

## 記事を追加したあと

dev serverが起動中の場合、ファイルを保存すると自動でホットリロードされます。  
ブログ一覧（`/blog`）と個別記事ページ（`/blog/[slug]`）の両方に反映されます。

デプロイはPushしてCIが走れば自動で反映されます。

## ドラフトにしたいとき（まだ公開したくない）

現時点では下書き機能はありません。公開したくない記事はファイルを作らないか、ファイル名の頭に`_`を付けてAstroのコレクション対象から外す方法が使えます（ただし`_`始まりのファイルはAstroが無視するため、slugには使えません）。

シンプルに「書き上がったら追加する」運用が一番楽です。
