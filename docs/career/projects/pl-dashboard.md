## 2. 採用担当目線で欲しい情報 (質問リスト)

### ★★★ 必須級

1. **ダミーデータでの画面キャプチャ**
   - profile.md: 「ダミーデータでのスクショ・デモあり」と書かれているが、サイトには未掲載
   - PL ダッシュボードの主要 3 画面 (一覧 / 詳細 / 入力) をダミー化して掲載する許諾を取れるか
   - → 「Cloudflare D1 + OOUI 実績」の最強の証拠になる、第 2 / 第 4 希望に直撃

2. **Zero Trust 設定の中身**
   - Identity Provider に Google Workspace を選んだ判断
   - アクセスポリシーの設計 (誰が何を見られるか)
   - 監査ログをどこまで取っているか
   - → Cloudflare 実践者として希少なエッセンス

3. **OOUI を選定した判断軸**
   - 「OOUI で組んだ」と書いているが、なぜ OOUI なのか (タスク中心 UI との比較)
   - PL という業務領域における OOUI の相性
   - → 「デザイン理論派」の本人なら語れる、デザイン × エンジニアリングの結節点

4. **データ移行と運用**
   - Excel から D1 へどう移したか (一括 import / 並行運用 / 手作業移管?)
   - 先輩 (主ユーザー) のリスクキリングをどうしたか
   - 月次更新は誰がやっているか / 自動化されているか
   - 数字: ユーザー定着率 / 更新頻度

### ★★ あると差別化される

5. **「自発的に提案・実装」の経緯**
   - Excel が見にくいと判断した瞬間 (上司から指示されたわけではないはず)
   - 提案までに何を準備したか (試作品? モック?)
   - 上司の OK までの流れ

6. **D1 のスキーマ設計**
   - PL の科目構造を SQL でどう表現したか
   - 売上 / 費用 / 利益のドリルダウンをどう設計したか
   - 月次比較・前年同月比のクエリパフォーマンス

7. **Workers + Hono 構成の理由**
   - クライアントワークでも使える社内テンプレ化したか
   - 認証は Better Auth? Zero Trust が前段にあるなら不要?

8. **失敗 / 想定外**
   - 「Excel の方が良かった」と言われた瞬間 (もしあれば)
   - パフォーマンスで詰まった点
   - データ整合性で詰まった点
