## 2. 採用担当目線で欲しい情報 (質問リスト)

> **状況**: 現状はアーカイブのテキスト説明のみで**画像が一切無い**。site-improvement-plan.md でも「中途半端が一番悪い」と書いている通り、ここは早急に強化候補。Round 2 voice memo: 「ロゴデザイン作品 3〜5 点は出せる」とのこと。

### ★★★ 必須級

1. **公開可能なロゴ作品 3〜5 点の選定**
   - Swept のロゴは確定 (mdx に詳細あり)
   - AIChatClip のロゴ (現状サイトで使用済み)
   - その他、クライアントワーク・自主制作・大学時代のもの含めて 3〜5 点
   - クライアント情報を伏せる必要があれば「飲食店 (兵庫県)」程度の抽象化で OK
   - **ファイル形式**: SVG (拡大しても綺麗 / Astro Image と相性が良い) を推奨

2. **各作品 1 段落ずつのキャプション**
   - 何のための / どんな課題を解いたロゴか
   - 形 / 色 / タイポグラフィの判断軸 1 つずつ
   - Material Design や設計理論との接続点 (本人が理論派と主張するなら必須)
   - → 「雰囲気でデザインしてない」証拠の本丸

3. **Design Works ページのレイアウト方針**
   - 現状はテキストのみ → 画像ギャラリー型 (Masonry / Grid?)
   - 各作品クリックで詳細モーダル / 別ページ展開のどちらにするか
   - サイト全体の PRTS テイストとの整合 (背景・余白・フォント)

### ★★ あると差別化される

4. **印刷物 (DTP) の作例**
   - 名刺 / ポスター / チラシ / 冊子のうち入稿実績があるもの
   - 印刷会社・用紙・特色印刷などのこだわり点
   - DTP 入稿で詰まったエピソード 1 つ

5. **UI モック作例**
   - Figma のモック画像を 2〜3 点
   - 実装に至った / 実装前で止まった、両方含めて

6. **Visual Identity 一式 (ロゴだけでなくブランドガイド全体)**
   - 配色システム / タイポグラフィ / アイコンセット
   - 1 件あればブランディング能力の最強の証拠

### ★ 余裕があれば

7. ペルソナ別 / 用途別のロゴ展開バリエーション
8. ロゴ採用までのスケッチ / ボツ案 (思考プロセスを見せる)
9. 印刷見本写真 (実物を撮影)
10. 自分が好きなデザイナー / 影響を受けた事例
11. グラフィックデザインのスキルと UI デザインのスキルの自己分離 (本人感: グラフィックデザイナーというより UI/UX デザイナー寄り)
