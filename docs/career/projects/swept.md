# Swept — 質問リスト

> サイトソース: `src/content/projects/swept.mdx`
> ライブ: https://swept.example.com (現状ダミー URL — 実 URL に差し替え必要)

## 1. 現状サイトに載っている内容

- 3 人チームの社会起業プロジェクトに約 2 年関わった (2024 — 2026.01)
- 役割: デザイナー → 実装まで担う Product Engineer への転換点
- Phase 1 (旅行・約 1 年): ユーザーインタビュー集中、ペインがターゲットに到達しないと判断して撤退
- Phase 2 (愛着・2025 春 — 2026.01): ロゴ / ブランド / 名刺 / Web の設計と実装
- 離脱理由: 福祉・社会起業寄りに色合いが強まり、自分のスタートアップ像とのズレが大きくなった
- ロゴ「Swept」: 過去のトラウマや蟠りを掃き清める由来。Sweep の動き / Connection の繋がり / Heal Green + Warm Gray
- DTP 入稿まで担当
- Astro + Tailwind で Web 実装
- 残メンバーで Web 運用は継続

## 2. 採用担当目線で欲しい情報 (質問リスト)

### ★★★ 必須級

1. **2 年間の役割の具体度**
   - チーム 3 人の他 2 人の役割は何だったか (CEO 役 / マーケ役 等、機微情報を出さない範囲で)
   - その中で自分が「これは絶対に自分が担った」と言える領域
   - 「これは他の人がやってくれた」と認めるべき領域
   - → 「2 年間で 1 人で全部やってた」と読まれない・読ませない設計

2. **Phase 1 撤退判断の中身**
   - ユーザーインタビューを何人 / 何ヶ月 / どんなプロファイルに実施したか (数字 1 つ)
   - 「ペインが到達しない」と判断した定量・定性のシグナル具体例
   - 撤退提案を自分から出したのか / チーム合議か
   - → 「ズルズル続けない」判断ができたエピソードを面接で語れる解像度に

3. **Phase 2 の最終成果物の具体**
   - ロゴ / 名刺 / Web のスクショや画像を**サイトに出せるか**
   - クライアント / Swept としての公開許諾は取れる前提か
   - 出せないとしたら、もしくは部分的に伏せ字にして出せる範囲

4. **離脱判断のサイトでの語り方**
   - 現状 mdx は「方向性の違いを話し合った上で離脱」と書かれている
   - 採用担当が「すぐ辞める人なのでは」と読むリスクへの先回り
   - 起業仲間の学歴 (京大卒・灘卒) には触れない (profile.md 既知)
   - 「福祉・社会起業の方向性」は具体性ある記述だが、Swept 側の現メンバーが見たときに不快ではないかチェック

### ★★ あると差別化される

5. **ユーザーインタビューでの具体テクニック**
   - 「沈黙の使い方」 → 何秒沈黙すると本音が出る、等の実践知
   - 「定性と定量の切り分け」 → 具体的にどう切り分けたか
   - 「カスタマージャーニーマップによるペイン定義」 → 1 個のペインを抽出した具体手順
   - → 起業準備で身についたものとして Bio に既に書いているが、Swept ページに落とし込むと厚みが出る

6. **デザイン → 実装に振り直したとき直前と直後の自分**
   - 初めて自分の設計を自分で実装した瞬間に何を感じたか
   - 「これは行ける」と手応えを掴んだ具体的瞬間 (1 つ)
   - 当時の AI クオリティの荒削りさをどう乗り越えたか

7. **ロゴの意図の根拠**
   - Sweep の曲線・Connection の一筆書きを **Material Design 2 / 3 の理論** とどう接続させたか
   - Heal Green と Warm Gray の RGB / HSL / 心理学的根拠
   - → デザイン理論派の本人なら語れる、Swept ページで強い差別化になる

8. **Astro 採用の判断根拠 (当時の意思決定)**
   - 当時 (2025 春) の候補は Next.js / Astro / Nuxt / SvelteKit?
   - SSG の理由 / SEO 要件 / コンテンツ更新頻度の見立て

### ★ 余裕があれば

9. Swept の URL: 現状 mdx に `https://swept.example.com` (ダミー) → 実 URL を出すか / 出さないなら何と書くか
10. Phase 1 のテーマ「旅行」の仮説の中身 (どういう旅行課題を解決するつもりだったか、抽象化レベルで)
11. Phase 2 のテーマ「愛着」へ転換した時のチーム合意プロセス (撤退と新テーマ提案を誰がリードしたか)
12. 名刺デザインで意識したこと / DTP 入稿で詰まったこと
13. 自分が離脱した後の Swept 現状 (再就職先で説明可能な範囲で)
