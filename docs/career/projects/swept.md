## このリストの方針

ブログの最近のノリ (`just-ship-it.md` / `vibe-coding-and-designer.md`) に寄せて書き直しました。

- 2 年関わって離脱した過去のプロジェクトなので、「振り返ってどう思うか」「あの時何を感じてたか」を中心に拾う
- 数字 / 役割の切り分け / 離脱理由の語り方は、聞かれて困らない解像度で残しておきたいので `[ちゃんと]`
- ロゴ / デザインの根拠は、デザイン理論を読み込んできた本人が語れるところなので `[ちゃんと]` 寄り
- それ以外は気楽に「あの時こうだった」「今思うとこう」レベルでOK
- 一気に全部答えなくていい。気が向いたものから音声メモで返してくれれば順次反映します
- Swept の現メンバーが見ても気まずくない範囲、というのは常に意識する

各質問に `[ちゃんと]` / `[ゆるくでOK]` を付けてあります。

### 音声メモ #1 (2026-05-18) — 質問 1〜27 まとめて回答

書き起こし全文: `voice-memos/swept-1.md`

主な確定事項 (mdx / ブログ反映用にここで要点だけ):

- **チーム役割**: 3 人チーム = 自分 (デザイン+実装) / CEO 役 / マネージャー役。事業企画・リサーチは 3 人で。対外プレゼン・コミュニティ交流は他メンバー。
- **時間 / 報酬**: 副業的に週 3 日相当、完全ボランティア。離脱時に成果物の対価として経費分のみ精算。
- **インタビュー数**: 2 年通算 100 人超。海外旅行客にはアポなし突撃 (浅草・空港)、国内ターゲット切り替え後はパーティー等で実施。
- **Phase 1 撤退の決め手**: CEO 役の中の違和感 + ニーズが想定内 + 自分ごと化できていない、で合意。マネージャーが最初に切り出し、サクッと決まった。
- **Phase 1 で試したテーマ 3 つ**: (a) 骨伝導 × 観光 (シーズ発信で失敗) → (b) 外国人観光客の荷物 (異国サービスへの不信が壁) → (c) 国内旅行の後片付け (実装するとコインランドリー的になる、で撤退)。
- **沈黙の効用**: バイアスがかからず本音が出る。新人研修等でも使う。
- **CJM**: 個人作業より「チームの目標言語化」に効くツールという所感。「旅行荷物 → 実は後片付けがペイン本丸」を抽出できたのは CJM のおかげ。
- **「愛着」転換**: 旅行撤退時に自分から「愛着は人生のど真ん中」と提案、CEO 役の「お前が納得するならそれしかない」で決まった。
- **Astro 採用理由**: Cloudflare 相性 + 当時の JS への抵抗 + コーポレートサイトにオーバースペック不要、で Astro。今もう一度選んでも第 1 候補は Astro。
- **ロゴの根拠**: 黄金比の円のみで構成 = 再現性。メタファーは「掃き清める」ではなく「愛着」。**卵 (守るべきもの / コロンブスの卵 / 我々はまだ卵)** + **S** + **抱擁** を組み合わせ。1 年目に作ったグラデ強めのバージョンを SNS 耐性のあるシンプルな形に作り直し。
- **マテリアル**: ブランド設計には未使用、サイト制作のカラー・コンポーネントで使用。
- **BI 設計の作法**: キーワード抽出 → マインドマップ。これ無しでは進めない。
- **「Product Engineer 自覚」**: 2025 末、Cloudflare 含むデプロイまで一連を完結できると気づいた瞬間から。
- **離脱の温度感**: 「方向性の違い」は唯一正しい。スタートアップ像のズレ、事業の方向、コミュニティ寄りに振れた点が要因。「福祉・社会起業寄り」表現は OK。
- **すぐ辞める懸念への返し**: 「プロダクト中心の磨き込みに集中する企業なら自分は残れた」が言語化済の離脱基準。最初から「長くはおらん」と表明済で、認識すり合わせは事前にできていた。
- **離脱後**: 残メンバーで運営継続、シェアハウスで一緒に住んだまま情報共有も続く。
- **学び**: 「ボランティアで、自分が本当に解決したいこと以外はやらない」「数年後そこにいる自分のイメージがつくか」が次の判断軸。Swept では明確な年数は決めず「数年後いなくなってても、まあそうだよね」「とはいえ 1〜2 年はいるやろ」を 3 人で共有していた、結果ちょうど 2 年で離脱。

確認したい・後で深掘りすべきもの:

- 質問 15 (Heal Green / Warm Gray の RGB / HSL の具体値、何案試したか) — 次回
- 質問 17 (初めて自分の設計を自分で実装した瞬間の具体エピソード) — 次回
