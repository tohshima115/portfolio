## このリストの方針

ブログの最近のノリ (`just-ship-it.md` / `vibe-coding-and-designer.md` あたり) に寄せて書き直しました。

- 「採用担当に刺さるか」より「自分が後から読み返して面白いか / 記録として残しておきたいか」優先
- 数字とか設計判断はちゃんと書きたいので、そこは引き続きちゃんと聞く
- それ以外は「なんでそう思った?」「どこでつまずいた?」みたいな、エピソードベースで拾う
- 一気に全部答えなくていい。気が向いたものから音声メモで返してくれれば順次反映します

各質問に `[ちゃんと]` / `[ゆるくでOK]` を付けてあります。
