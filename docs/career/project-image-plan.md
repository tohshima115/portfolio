# プロジェクト別・必要画像の整理

ポートフォリオサイトに載せる画像の洗い出し。採用文脈で「実力の証明」「信頼性の担保」を上げるための画像を、プロジェクトごとに優先度付きでまとめる。

## 前提：ポートフォリオの画像がやる仕事

採用文脈で画像が果たすのはだいたい 4 つで、これごとに必要な絵が変わる。

| 役割 | 何を証明するか | 効くシーン |
|---|---|---|
| **Shipping Proof** | 「実際に世に出てる」を疑う余地なく見せる | ストア掲載・ライブ URL・実物写真・課金画面 |
| **Craft Proof** | 細部のクオリティ。ピクセル・タイポ・余白・色 | クローズアップ・並列比較・ブランドガイド |
| **Thinking Proof** | 設計判断・なぜそうしたか | アーキ図・before/after・OOUI 切り替えの比較 |
| **Self Proof** | 「この人」が作っている空気 | 作業風景・スケッチ・物理現物の手元写真 |

文章だけだと「言ってるだけ感」が残るので、**主張ごとに 1 枚画像を当てる** くらいの密度がほしい。

## AIChatClip

ライブで動いてて課金まで来てるのに画像ゼロは本当にもったいない。ここは厚めに。

### Shipping Proof（必須帯）
- **Chrome Web Store の掲載ページ** — ストアバッジ＋掲載スクショ。これだけで「審査通っている」は一発
- **Firefox Add-ons の掲載ページ** — Chrome と並べて 2 つ揃うと強い
- **拡張機能が ChatGPT 画面上に乗ってる状態** — プロダクトの存在を瞬時に伝える hero
- **「ChatGPT → クリップ → Obsidian に届いた」の連続スクショ or GIF** — 価値が 3 秒で伝わる
- **Remotion 製プロモ動画の埋め込み or GIF 抜粋** — 自作してるので証拠にもなる

### Craft Proof
- **拡張機能ポップアップ UI のクローズアップ** — タイポ・余白・状態（hover / loading / done）
- **Web の LP（aichatclip.com）スクショ** — Tailwind v4 + Motion でどう仕上げたか
- **設定画面 UI** — 一番触られる画面なので craft が問われる場所
- **AIChatClip 自体のロゴ・アイコン** — extension アイコンが小さく効くプロダクトなので独立カットがあると良い

### Thinking Proof（ここが他と差がつく）
- **アーキテクチャ図（クリーンな SVG 版）** — 現状 ASCII art なので、図にすると一気に「設計できる人」感が出る
- **「Embedding で決定論的に振り分け」の概念図** — 一番ユニークな技術判断。LLM だとブレる → Embedding で固まる、を図 1 枚に
- **WebSocket + Hibernation API の通信図** — Durable Objects がなぜ要るかの説明補助
- **「完了アイコン問題」の before / after** — UX 改善で離脱が減った話を絵で。これがあると「数字とデザインの距離が近い人」が伝わる
- **登録ユーザー / クリップ数の成長チャート** — 51 人 / 週 41 クリップを可視化

### あると差別化
- **オンボーディングフロー全画面** — ユーザーテストで磨いた経緯ありなので絵で見せる価値あり
- **拡張機能のアイコン配置ロジック修正の before / after**

## デザインの仕事

「デザインができる」を主張するページに画像がないのは矛盾レベル。最重要。

### Shipping Proof
- **3 ロゴ全部の最終形（SVG / 高解像度）** — ニューセンチュリー / 江東ウォーキング / 不思議の国のアリス杯
- **ブランドガイドライン PDF or 抜粋ページ** — ロゴ単体ではなく「通した」唯一の事例なので、ここを見せないと意味がない
  - 配色パレットページ
  - タイポグラフィルール
  - ロゴ使用 OK / NG 例
  - ミニマムサイズ規定
- **入稿実績ありの名刺の物理写真** — 手元・テーブル置き・名刺入れから出ている状態など 2〜3 カット
- **ポスター現物の写真** — 物理物が手元にあるなら撮影。壁掛けや手持ちなど環境込みだと尚良
- **チラシ（ブルーベリー愛犬ファミリー協会）** — 現物 or 入稿データ

### Craft Proof
- **ニューセンチュリーロゴの「犬モチーフ」拡大カット** — 本文で語ってる決め手を絵で示す
- **ロゴ展開バリエーション** — モノクロ / 縦組 / 単色 / 反転など使い分け
- **配色の比率図解** — 「ベース / メイン / アクセントの比率」を本文で挙げてるので、自作品でその比率を図示すると説得力が爆増する

### Thinking Proof
- **ロゴのスケッチ → ラフ → 詰め → 最終** の 4 段階くらい並べた process shot（1 案件分でいい）
- **「具体的修正指示」を回避した話の前後のラフ** — もし出せるなら
- **Material Design の「型」を踏まえてどこをずらしたか** の比較図

### Self Proof
- **手書きスケッチノートの写真** — 「ヒアリングで沈黙使う」「裾を見にいく」と語っているので、現場でメモしてる人感が出るカット

### 低優先
- UI モック（本人が「練習がてら」と書いてるので無理に出さなくていい）

## Swept

hero ありなのでそこは OK。あとは厚みが足りない。

### Shipping Proof
- **ロゴ最終形** — 主な成果物リストの筆頭なのに見えない
- **ホームページのスクショ**（フル or ファーストビュー）
- **名刺の現物写真**
- **診断ツールのスクショ** — 成果物リストにあるのに描写なし

### Craft Proof
- **ブランドアイデンティティのボード** — ロゴ + 色 + フォント + 適用例を 1 枚に
- **「愛着」テーマに切り替えてからのキービジュアル**

### Thinking Proof
- **Phase 1 → Phase 2 のテーマ切り替えを示すスライド or 図**
- **CJM（カスタマージャーニーマップ）の実物 or 再構成版** — 「後片付けが本丸」と気づいた絵
- **ユーザーインタビュー集計の一部** — 100 人超のインタビューの片鱗

### Self Proof
- **3 人で集まって作業してる風景写真**（シェアハウスでやってた話の補強）— 顔が出るのが NG なら手元やホワイトボードでも

### あると差別化
- **対外プレゼンスライドの抜粋**（ロゴ・名刺・Web と並ぶ成果物のひとつなので）

## 業務ツールいろいろ（Web 制作会社）

クライアント情報が絡むので「ダミー差し替え」前提の絵作りが必要。

### Shipping Proof
- **PL Dashboard のスクショ（ダミーデータで）** — 5 個の中で一番画面映えする
- **Schedule Distributor Electron アプリの UI**
- **議事録自動生成の Obsidian 上の見え方** — MD ファイルが NAS 経由で配られている結果

### Thinking Proof（このページはここが本丸）
- **PL Dashboard の OOUI 切り替え before / after** — 「アクション起点 → 勘定科目起点」に組み直した話を絵で。これ 1 枚で「ただ AI に作らせた人じゃない」が伝わる
- **議事録ツールのインフラ構成図** — NAS マウント + Ubuntu + 社内 GCP Gemini という制約の中での設計を図にすると「制約下で組める人」が一発で示せる
- **スクレイピング自動化フロー図** — 参考サイト → Python + Docker → FTP の流れ
- **Stack Modernization の比較表（ビジュアル版）** — Before / After を写真や UI スクショ付きで

### Craft Proof
- 低優先（業務ツール文脈なので craft より thinking が読まれる）

## ページ全体・横断で効くもの

個別プロジェクトじゃなくサイト全体の信頼性に効くやつ。

### About / Hero 周り
- **本人のアバター画像 or 顔写真** — 採用文脈だと「人」が見えるとぐっと進む。顔出し NG なら横顔 / 後ろ姿 / 手元のシルエットでも
- **作業デスクの写真** — Cloudflare 主戦場と言うなら、その作業環境（モニター・ツール）が見えるとリアリティが出る

### スキル / 信頼性
- **Google UX デザイナー認定証の画像** — 持ってるなら出すべき
- **使用スタックのアイコン一覧（Cloudflare / React / Tailwind 等）** — 「Cloudflare 主戦場」を視覚化

### プロセス感
- **手書きノート / スケッチブックの写真** — 「考えてる人」の証拠
- **Obsidian の Vault 画面** — 本人の知的生産環境（aichatclip / 業務ツールでも繰り返し登場）

## 撮ったり作ったりする優先順位

サイト全体の説得力をブーストする順：

1. **AIChatClip のストア掲載 + 連携 GIF + アーキ図 SVG** — ライブプロダクト × 3 種の証拠
2. **デザインの仕事のロゴ全点 + ブランドガイド + 名刺現物写真** — 「デザインができる」の現物
3. **アバター / About 用の人物画像** — 全ページに効く
4. **PL Dashboard の OOUI before/after** — 業務ツールページに「設計判断ができる人」を一発で乗せる
5. **Swept のブランドボード + Web スクショ + 名刺**
6. **手書きスケッチ系・作業環境写真**（process / self proof）
7. **その他クローズアップ・図解**
