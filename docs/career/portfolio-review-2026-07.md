# ポートフォリオ現状評価 — 2026.07.09 時点

**評価の前提**: 応募先が公開している「落とすポートフォリオ」の基準を裏返し、以下2軸で採点した。

1. **ビジュアルの作り込み** — 浅い / 甘いと判断されないか
2. **実績の言語化・制作意図** — 意図が読めない / 言語化が弱いと判断されないか

見せたい像は **「デザイナー寄りのフロントエンド(実態はバックエンドも含む Product Engineer)」**。したがってデザインとエンジニアリングの両面が評価対象。

**採点方法**: `pnpm dev` で全ページをモバイル(390×844)/ PC(1440×900)両方で実機確認し、コンテンツ(`src/content/`)を読んだ上での評価。

---

## 総評

> **言語化は合格ラインを超えている(8/10)。ビジュアルは「トップの世界観は強いが、証拠となる絵が皆無」という片肺状態で 6/10。**

危ないのはビジュアル側。ただしその中身は「装飾が足りない」ではなく **「実物(スクショ・ロゴ現物・動画)が1枚もない」** こと。
→ Works に素材を追加する方針は、優先順位として完全に正しい。

---

## 軸1: ビジュアルの作り込み — 6/10

### 効いているところ

- トップのグラデーション Hero は記憶に残る
- Grotesque 系の極太見出し + monospace ラベル + 生成り背景というシステムが**全ページで一貫**している。Works 詳細 / Blog 一覧 / Contact までトーンが崩れていないのは、テンプレ組み合わせ勢と明確に差がつく
- Blog サムネイルが緑系イラストで統一され、「継続運用されているサイト」に見える

### 弱いところ(致命的な順)

#### 1. Works 詳細に画像・動画がゼロ 🔴

AIChatClip のページは、プロダクトの UI を一度も見せないまま DECISIONS に入る。
**「デザイナー寄り」を名乗る人のポートフォリオで、デザインした画面が1枚も出てこないのは最もわかりやすい減点。**

#### 2. Design Works が完全にテキストのみ 🔴

ロゴ3件・名刺・ポスターと書いてあるのに絵がない。`public/DesignWorks/` には SVG が2枚あるだけ。

判断基準の言語化(余白と情報密度 / 曲で例えるヒアリング / 型なしと型破りは違う)は**すでに非常に良い文章**なので、
**現物画像が付いた瞬間に、このページはサイトで一番「デザイナー寄りエンジニア」を証明するページに化ける。**
→ Works の動画より先でもいいくらい。素材出しから必要。

#### 3. アーキテクチャ図が ASCII アート 🟡

`src/content/projects/aichatclip/architecture.mdx`。モバイルだと横スクロールになる。
エンジニア単独なら ASCII は愛嬌だが、デザイナー寄りを名乗るなら「図解が美しい」は見せ場のはず。
サイトのデザインシステムに合わせた SVG 図に置き換える価値が高い。

#### 4. トップのスクロール体験にデッドゾーン 🟡

モバイルで全長 約12,500px。Works と Blog の間に「ほぼ白紙の画面」が丸ごと挟まるスクロール帯がある。
また遷移中に Swept の上に AIChatClip のテキストが透けて重なる瞬間がある。
演出自体は良いので、**遷移ゾーンの距離を詰める**調整で印象が締まる。

## 軸2: 実績の言語化・制作意図 — 8/10

採用側の「意図が読めない」という落とし基準の**真逆**を行けている。

- **DECISIONS の「問い → 結論先出し」構造**(例: 「Q. なぜこのロゴにしたのか → A. アイコン列で浮かないことを最優先にしたから」)は、審査する側が一番読みたい形そのもの
- 数字が具体(登録51人 / 有料1人 / インフラ費ほぼ0円 / 週41クリップ)。**盛っていないのが逆に信頼になる**
- **失敗と撤退が書いてある**(Swept の撤退、Firefox のスパム判定、LLM のブレ → 埋め込みへの切替)。「今なら変えること」の章はシニアの振り返りの書き方

### 減点2点の内訳

- (a) 意図は読めるが **証拠(ビジュアル)が添付されていない** — 軸1と同根
- (b) AIChatClip の `web.mdx` / `promo.mdx` が draft のまま。表示からは除外されているので公開自体は可能だが、
  **「LP・マイページの設計意図」はデザイナー寄り訴求の本丸**なので優先的に書く価値がある

## 計画外で「ここもやった方がいい」提案

### 1. 【即修正・バグ】OGP 画像が 404 🔴

- `src/layouts/BaseLayout.astro:17` が `/og-image.png` を指しているが、`public/og-image.png` が**存在しない**
- 採用担当が Slack や X で URL を共有した瞬間に第一印象が壊れる。Works 素材より先に1枚置くべき
- ついでに `BaseLayout.astro:49` の JSON-LD `sameAs` が `https://x.com/`(プレースホルダ)のまま

### 2. Contact に稼働ステータスがない 🔴

現状は匿名フォームだけ。就活文脈の導線(GitHub / X / 面談歓迎)が主役になっていない。

```
[STATUS] 2026年7〜8月に現職退職予定 / 9月以降入社可能 / カジュアル面談歓迎
```

サイトを採用フィルタとして使うなら、この1行が最重要情報。

### 3. 経歴がモーダルの奥に隠れている 🟡

`AboutDeepDive.tsx` の「経歴 / Career Path」はクリックしないと出ない。採用担当は流し読みするので開かれない可能性が高い。

- 年表サマリー(2023 起業準備 → 2025 デザイン事務所 → 2026 個人開発)は**インラインで見せる**
- 詳細だけ Deep Dive に残す

→ About 構成に感じている違和感は、おそらくここが原因。

### 4. AIChatClip の `web.mdx`(LP・マイページの設計意図)を最優先で執筆 🟡

スクショを入れる作業と**同時にやる**と、画像に意図が紐づく形で書けて一石二鳥。
`promo.mdx`(Remotion)は動画を貼れば文章は短くて済む。
