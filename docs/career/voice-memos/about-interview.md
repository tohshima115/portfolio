## ストーリーの背骨

**経営→起業準備→デザイン→個人開発を要約すると**
自分の力試しをずっとやってきた、という感じ。受験そのものが楽しかった性格で、自分が勉強したものがどれだけ世界に通用するかを試すのが楽しかった。経営も起業準備もデザインも個人開発も、ある程度通用しそうだなというところまで行けるかどうかを確かめながらやってきた。

**転職動機（本音）**
現職は社長と自分の「経営に対する宗教」が合わない。社長は営業畑出身で、新規事業を思いつきで出して開発側が振り回される構造になっている。加えて会社業績への懸念、Web デザインが「ツールを使える人」枠になっていてデザイナーとして能力を発揮できていない自覚もある。
→ これはサイトには書かない。

**転職動機（建前）**
Web デザインから設計寄り・開発寄りに重心が移ってきた。ちいさい会社でどんどん任せられる領域が広がった結果、今はほぼエンジニア寄りのことをしている状態。そっち側にスキルが寄ってきているので、最初から開発寄りの場所を狙った方がいいと感じた。これは嘘ではない。

**3ヶ月で正社員退職**

ポートフォリオサイトに載せることに意味があるのか疑問。文章で言えるものじゃないし、面接での受け答えやサイトの内容で見えてくるもの。わざわざ書いておく必要はない。

**起業準備 2 年で得たもの**

- ユーザーインタビューの度胸
- 小さいスパンで検証することへの抵抗のなさ・検証に時間をかけることを厭わない感覚
- 経営者視点（それは本当に必要なのかを上流から問い直す目線）
- 頭でっかちだったのが、まず動いてみる側に振り切れた

## 同年代との差別化軸（3つ）

1. **経営の上流から見られる**: 完璧より出すが大事というのを肌で感じられている。その上でエンジニア視点での保守性は最低限クリアする意識を持っている
2. **幅広いものづくり**: 動画（Remotion）、画像生成、デザイン、マーケティング（ロゴ・ブランディング）まで一通りやってきた。AI 時代の「1人で完結する幅」と相性が良い
3. **アーリーアダプター気質**: 新しいものはまず触る。アプリ・Web の登録ハードルが極端に低く、UX をリバースエンジニアリング的に分解する癖がある。スイッチングコストを雑に見積もって積極的に乗り換えていく

## デザイン実務でエンジニアの盲点を埋めた例

- **ニーズ検証・課金検証を先にやる**: AIChatClip は MVP 着手前に 1,000 円課金意思確認。「必要じゃないものを作ってしまう」エンジニアにありがちな失敗の回避

## Cloudflare の語り方

**使い始め**
2025 年夏過ぎ頃。ほぼ 1 年経過（2026-05 時点）。

**スタンス**
主戦場、ほぼ手癖で選んでいる感覚。

**なぜ気に入っているか**
- 個人開発者・スタートアップにここまで優しいサービスは他にない
- SQLite ベースの D1 が React Router v7 / Astro と素直に噛み合う
- Vercel と違って「なぜ動かない / なぜ動くか」が自分で追える難易度
- SSR / SSG / CSR の境界が、Vercel ではぼやけていたものが Cloudflare に載せた瞬間に明確になった（= 訓練として効いた）

**他の環境**
- AWS: 触ったがコスト構造が見えず投げ出した
- GCP: Google アカウント連携でスポット利用、ホスティング経験なし
- ホスティングは現状 Cloudflare 一択
