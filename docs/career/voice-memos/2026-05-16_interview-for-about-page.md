### B. Cloudflare の話

**4. 初めて触ったとき「あ、これいいな」と思った瞬間は？**
「今でも無料でこんなにできんの？」という感覚。色々試してもそんなにお金がかからないから、ビクビクする必要がない。マジで金かからずずっと遊べる、やりたいこと何でもできるじゃんという感じ。「大人のおもちゃ」（アダルトな意味じゃなく）みたいな感覚。

**5. D1 を初めて使ったときの感想を一言で言うと？**
「むず」。バックエンドをまだよく分かっていない段階でやっていたのもあって、うまく使えなかった。（後から振り返ると、マイグレーションをちゃんとやっていなかっただけだったかもしれない。）何回かやるうちに慣れてきたら「めっちゃ便利」に変わった。SQLite ベースで分散型、ランニングコストが安い、というのが個人開発・新規事業フェーズにはすごくありがたい。トラフィックが増えた時の対処より、まず事業を継続できるコスト構造の方が大事だと思っている。

**6. 「ここはまだ微妙だな」と思っている部分は？**
正直なかなか思い当たらない（気づいていないだけかもしれない）。強いて言えば：
- Next.js との相性があまり良くない（Vercel の方がそこは強くて当然だけど）
- CMS 系をどうするか問題。Cloudflare だけでは完結しないケースがある。最近 WordPress の精神的後継みたいなものを出したのは知っているが、本番投入するまでには至っていない

**7. もし Cloudflare が明日なくなったら次は？**
GCP（Google Cloud）かな。今更 Vercel に戻る気はない。Cloudflare の「全部盛り」感が好きだったので、同じような体験ができるものを探しに行く。Cloudflare はデータベース、AI、ストレージ（R2）、メール（Email Routing + Email Send）まで個人開発に必要なものがほぼ全部揃う。これを乗り換えるのはなかなか大変だと感じている。

### D. 性格・癖・こだわり

**12. やりすぎだなと思う癖・習慣は？**
自覚がないだけかもしれないが、強いて挙げると「成長投資にお金を糸目なく使う」こと。それで貯金がない。ただ「最高スペックを買う」わけではなく、「自分の要求水準にジャストフィットするものを買う」という感じ。オーバーエンジニアリングが嫌いで、動画とメールしかしないのに iPhone Pro を買っている人を見ると1時間問い詰めたくなる（笑）。逆に自分では「買ったけど使ってないものがほぼない」のはこだわり。お金が増えても多分そのまま気にし続けると思う。

**13. 「これがないと無理」という環境・ツールは？**

- Claude Code
- 自作キーボード（Roba 分割キーボード + 大西配列 + トラックボール付き）。手をキーボードから離さずに作業が完結する
- 複数画面（最近なんとかなるかもとは思い始めているが、まだないと辛い）

**14. 一人黙々とやる時間と誰かと話しながら作業、どっちが好き？**

### F. 「自己紹介」のイメージ

**20. アークナイツのキャラクタープロフィールの「どこ」が好き？**
情報と構造。最初に名前・生年月日・出身などが淡々と並んでいて、次に2〜3行の好みや趣味、その次にスタック的な項目（技術スタック的なもの）、最後に一言（「コーヒーが好き」みたいな）。そこしか読まないかも、というくらい自分的にはあの構造が好き。

**21. 自分の「キャラクター設定」を3行で書くとしたら？**
行き当たりばったり。思いつきでバッと動いて、毎回変な場所にいる。大学を卒業してから好き勝手やるようにしていったら、気づいたら社会のレールから外れていた。ただ衝動のままに動いているときが一番前に進める気がしているので、これからもそうしていくつもり。

**22. 「このページを見た人に自分をどう思ってほしいか」を言葉にすると？**
距離を感じてほしくない。「こいつオモロそう」とか「なんか頑張ってるな、可愛い奴やな」くらいの感じが理想。「舐めてほしい」（距離感を縮めてほしい）という感覚に近い。

### G. 今・これからの話

**23. 仕事以外で一番時間を使っていることは？**
個人開発（サイトブロッカーを自作中）。それ以外ではパートナーとの時間。YouTube ショートにも時間を使ってしまっている（反省）。週1日はノルマなしで自由にインプットする時間を作ろうとしている。AI のせいで技術の変化がとにかく速いので、「おもろそう」だけで新しいものを見に行く時間を意識的に確保したい。

**24. 最近「これ面白いな / 気になるな」と思ったことは？**
「文字」。多言語対応を考えていて、英語は左から右、アラビア語は右から左など、当たり前と思っていたことが全然当たり前じゃないと気づいている。RTL だと UI 構造そのものを逆にした方が自然になる、というのが面白い。ボタンの中に収まる文字数も言語によって全然違うのでレイアウトが崩れる問題もある。「文字って難しい」と思いながら AIChatClip とサイトブロッカーの多言語対応を考えている。

**25. 次に作りたいものや試したいアイデアは？**
サイトブロッカー。今使っているものへの不満:
- youtube.com をブロックすると music.youtube.com まで巻き込まれる（作業用 BGM が聴けない）
- 登録できるサイトが5件まで、それ以上は課金
- UI がとにかく悪い

コストがほぼかからないのに課金を強いられることへの腹立ちもあって、OSSで作ろうとしている。
