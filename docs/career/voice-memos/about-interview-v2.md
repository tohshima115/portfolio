## 「力試し」軸の解釈（重要: Round 1 から変更あり）

「飽きやすい・力試しが終わったら去る」と読まれるリスクについて聞かれて:

「力試し」で試したいのは現時点のスキルではなく、**諦めずに続ける力・粘り強く打ち込む力・継続する力**。受験が楽しかったのもそれを試すのが楽しかったから。「ある程度通用するかな」と感じたら次、という動き方に見えるかもしれないが、次の場所は**腰を据えて 10 年単位で関われる前提**で就活しようとしている。今のデザイン事務所を抜けるのはプロダクト寄りじゃなかったから、という理由だけ。プロダクト寄りの場所に行けば長く続く想定。

### 表現の変遷（同日中に 2 回改訂）

1. 最初: 「諦めずに続ける力がどこまで通用するかを試す」
2. 修正: 「諦めずに」が苦労・根性論ぽくて違和感あり → 削除
3. 確定: **「続けてきたものが、次のステージでどこまで自分の武器になるかを試す」**

「経営 → 起業準備 → デザイン → 個人開発」は切り捨てではなく積み上がり。それぞれで身についたもの（事業視点 / 検証の習慣 / 設計の理論）が今の個人開発の道具として残っている、という見せ方に。

## アーリーアダプター気質の具体例

- **Cursor / Antigravity**: 勃興初期から本番投入
- **Claude Code**: AIChatClip の設計時から課金開始（若干遅れたのは、当時やっていたのが Web デザインで Claude Code が必要な規模の開発をしていなかったから）
- **Affinity → Canva**: Affinity が Canva に買収されて無料化された日に切り替え
- **Aqua Voice**: 音声入力 Speech-to-Text サービス、課金継続中
- **自作キーボード + 大西配列**: 入力速度の上限を上げるための投資。AI 時代にアウトプット速度は上げた方がいいと感じて
- **Obsidian**: マジョリティより早めに始めた感覚
- **Material Design 2 / 3 (M3 Expressive 含む)**: 出た際に即追って読み込み

共通点: **スイッチングコストを雑に見積もって積極的に乗り換える**。いいと思ったものはとりあえず登録・試す。アプリ入れる・Web に登録するハードルが他の人より極端に低い。サービスを使いながら UX をリバースエンジニアリング的に分解する癖がついている。

## デザイン理論派としての自己認識

Material Design 2 のときからかなり読み込んでいて、3 が出たら 3 も。自分のデザインの基盤はここにある。

角丸の値・余白・コンポーネント設計に都度根拠を持てる程度には理論で組んでいる。「雰囲気でデザインしているわけではない」は一応主張したい。

「これいい / これダメ」を分解する癖はあるが、自社サービスへの反映はまだストック中段階。意識していること:
- **プライマリーボタンを必ず置く**（日本の Web で抜けがち）
- アイコンなしで文字だけは避ける

## Cloudflare 周りの本音

**失敗エピソード**
SSR / SSG / CSR で**何度も詰まった**。最近ようやく分かってきたが、まだ勉強が必要。ここは humble に書いていい。

**R2 / KV を採用していない理由**
単純に現状必要じゃないから:
- R2: アップロード型ストレージが必要になったら採用
- KV: キャッシュ層が要るタイミングで採用

選定眼の話。必要じゃないものは入れない。

**フロントエンド（優先度順）**

- 状態管理 ◎

- Rust（低レベルなコンピュータ理解として）
- パフォーマンス

- セキュリティ・パフォーマンス・脆弱性は人間が押さえるべき部分
- テンプレ的な書き方は AI に任せる
- 全体としてはバックエンド寄りに深めていきたい

## 業務改善ツール（カレンダー / LINE）の実態

**正体**: 音声 or テキスト入力 → AI が LINE 用文章 + カレンダー登録一覧を生成 → ボタンでカレンダーに登録する Electron アプリ

**作った理由**:
1. カレンダーと LINE への二重入力がだるい
2. 当時の AI のカレンダー自動入力精度が低かった
3. 業務報告フォーマットの細かい調整
4. 自分の勉強（Electron / GCP Google Calendar 連携）がしたかった

**本人の自覚**: 「今だったら AI チャットサービスだけで事足りる内容。設計ガチガチではなく勉強の側面が強い」
