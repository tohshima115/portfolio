# 2026-05-16 voice memo: 自己定義を「売り込み」から「個人ブログ」へピボット

> 採用市場向けに磨き上げてきた `about-copywriting.md` の路線そのものが、本人の実態とズレている、というところからの記録。
> 表現は voice memo の原文に近い形で残す（断定で書かない / 言い回しを上書きしない）。

---

## 1. 何が起きていたか

ポートフォリオサイトの肩書きを「Cloudflare で個人プロダクトを出荷している Product Engineer」と置いていたが、見るたびに **「自分じゃない人のもの」** を見ているような感覚があり、サイト全体に対してやる気が削がれていた。Round 1-2 で精緻に組み上げた肩書きほど、本人と乖離する違和感が強くなっていた。

突き詰めると「自分への自信のなさ」が震源で、文章のトーンや肩書きはその表層症状。

## 2. 本人の言葉 (要点)

### 売り込みそのものへの違和感

- 「売り込むという行為そのものにもズレを感じる。そんなに売り込むほどの…根本的な話をすると、全然自分に価値を感じてない」
- 「ほぼ未経験、出してるもんとかはあるしスキルもそこそこあるけど、言ってしまえば未経験のジュニアのエンジニアに価値あるんか、という感じ」
- 情報としては「そういう人を雇いたいところはある」と一定納得はしている。**ただ、個人的に「売り込めるもの」とは思っていない**。この乖離が大きい。

### 理想のトーン

- 作業ログに振り切れたら、結構ラクになる気はする
- 「自分の紹介」というより **「趣味の紹介」** の方に振り切った方がラク
- 「僕こんなのが好きで、こんなことしてます、よろしく」ぐらいのテンション = **自己紹介兼個人ブログ**
- それが面接や就職活動に通用するかはまた別問題、と本人も認識
- 今の「売り込む」空気は自分と相性がすごい悪い、と感じながらやっている

### 一番正直な実態

- 一番しっくりくるのは **「Cloudflare 愛好家」**
- 「Cloudflare が好きで、いろんな機能を試して実験して、それを仕事にしたいなと思っている人」
- 開発も課題解決も楽しい
- デザインは「アート寄り」ではなく **「工業的・UX 寄り・使いやすさ寄り」** が好き。「こうしたら使いやすくなるよね」を考えるのが好き
- ただし「めっちゃ強めです」とされると鼻につく → 「好きだなって思ってる」ぐらいの強度に留めたい

## 3. 確定した方針

| 論点 | 確定 |
|---|---|
| 自己定義 | **「Cloudflare が好きな 26 歳。デザイナー出身で、使いやすさを考えるのが好き。」** |
| トーン | 売り込みではなく、**個人ブログ / 趣味紹介**。「好きだなって思ってる」レベルの強度 |
| Round 1-2 路線 | **覆す**。「Product Engineer として売り込む」設計を捨てる。`about-copywriting.md` § 17 でピボットを宣言 |
| 反映範囲 | **TOP `AboutLayer` / `/about` ショート / `AboutDeepDive` Bio・Stance まで一気に書き換える** |
| Markdown 式の補足 | 維持。「淡々と何をしたかを示す」装置として、このトーンとは相性が良い |
| 「Product Engineer」という名乗り | サイト本体からは引っ込める方向 (見出し・dl ラベル等)。プロフィールカードやメタ情報の `roleLabel` 等で残すかは反映時に都度判断 |
| 業務改善 / AIChatClip 等の素材 | 削らない。**素材自体が勝手に「この人プロダクト作れる」と語る前提** で、上に乗る装飾だけ柔らかくする |
| 採用フィルタを通るかの懸念 | 通らない先には通らない、で割り切る。**続けられるサイトの方が、続かない完璧なサイトより 10 倍価値がある** |

## 4. 進め方

1. `about-copywriting.md` § 17 にピボット決定を追記（本ファイルへのリンク付き）
2. **TOP `AboutLayer.tsx` を新トーンで書き換え** → 本人レビュー
3. 肌触り OK が出たら、その語感を真として:
   - `src/content/_pages/about.mdx` (ショート About 本文)
   - `src/components/pages/about/AboutDeepDive.tsx` の Bio / Stance
   - その他 `roleLabel` / consistencyTable など、肩書きトーンが波及する箇所
4. 反映後にローカルで Astro dev サーバーを立ち上げて、TOP の About レイヤーと `/about` を実機チェック

## 5. 残しておきたい注意点

- 「Cloudflare が好き」を強めに出しすぎると、本人の感覚で「鼻につく」レベルに行く。**「好きだなって思ってる」の強度を逸脱しない**
- 「個人ブログ風」と言っても、**サイトの構造 / セクション分け / Markdown 装飾は維持**。書き換えるのはあくまでコピーのトーン
- 「採用に通用するか」の不安は本人にも残っているが、**続かなくなるリスクの方が大きい** ので、ピボット側に倒す判断
