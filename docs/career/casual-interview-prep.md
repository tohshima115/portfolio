# カジュアル面談 想定質問リスト

> **目的**: 初回のカジュアル面談 (30〜60 分、人事 or 現場マネージャ) で頻出する質問をストックしておき、回答を音声メモで残しつつ、サイト・職務経歴書・自己紹介と整合させる。
>
> **想定先**: Product Engineer / プロダクトエンジニア / コーポレートエンジニア / 業務改善エンジニア / Cloudflare 系開発、いずれもスタートアップ〜中規模企業のカジュアル面談。
>
> **使い方**: 答えに詰まる質問を選んで音声で回答 → `voice-memos/YYYY-MM-DD_casual-interview.md` に書き起こし → ここの該当質問の下に「回答メモ」として追記、もしくは `interview-prep.md` (今後作成) に転記。

---

## 1. 自己紹介・キャリア

カジュアル面談の冒頭 5 分で必ず聞かれる枠。1 分 / 3 分の 2 バージョンを用意しておきたい。

1. **自己紹介を 1〜2 分でお願いします**
2. **これまでの経歴を簡単に教えてください**
3. **大学では何を専攻していたか / なぜ経営学部だったのか**
4. **新卒入社の会社で何をしていたか / なぜ 3 ヶ月で退職したのか**
5. **起業準備をしていた 2 年間は具体的に何をしていたか**
6. **今のデザイン事務所での担当業務 / バイトリーダーとは何をしているか**
7. **デザイナー → エンジニアへ重心が移った理由は?**
8. **個人開発をはじめたきっかけは?**

> ✏️ ヒント: voice memo の「自分の力試しを続けてきた」軸で 1 本通すと一貫性が出る。

## 13. 答えに詰まりそうな質問 (要・回答練習)

voice memo を踏まえて、特に **準備しておくべき** と判断したもの:

- **9. なぜ今のタイミングで転職を?** → 「Web デザインから設計・開発寄りに重心が移ってきた。プロダクト本気でやれる環境に行きたい」をストレートに。
- **11. 今の会社の不満は?** → 社長の宗教違いを直接言わない。「自社で扱う Web デザイン領域では身につくスキルの伸びが頭打ちになってきた」レベルで止める。
- **15. 当社のどこに興味を?** → 応募先別のカスタマイズが必要。「公式ブログ / プロダクトを使った感想」を 1 つは持っておく。
- **63. 同年代との差別化は?** → voice memo 3 軸 (経営の上流から / 幅広いものづくり / アーリーアダプター気質)
- **66. 失敗エピソードは?** → 起業準備の登記前離脱 or Python 授業苦手意識。後者の方が安全。
- **77. 5/10 年後**: → 「開発の専門性を深めたい。ただし AI 進化で求められるものが変わる前提で柔軟に」
- **82. 希望年収**: → 「現職 200 万 → 350〜450 万を希望、伸びしろ次第で柔軟」

## 14. 反映フロー

1. このリストから「答えに詰まる」「言語化したい」質問を選ぶ
2. 音声で回答を録音 → `voice-memos/YYYY-MM-DD_casual-interview.md` に書き起こし
3. 当該質問の下に「回答メモ:」として整理した要点を追記
4. 応募開始前に **13 章のリスト分は最低限ロジックを通しておく**
5. カジュアル面談を経験するごとに、想定外だった質問・刺さらなかった回答を **15 章 (将来作成) / 履歴** に追記して更新
