## なんで作ったか

AI チャットで「これ良い回答だな」と思っても、放っておくと履歴の海に沈んでいく感じがずっと気持ち悪くて。Obsidian や Notion をメインの知識ベースにしているなら、そっちに流し込めればいいだけのはずなのに、その「最後の 1cm」が地味に面倒くさい、という個人的な不満が起点でした。

最初は **n8n で自分用に組んでいたパイプライン** で、サーバー内で完結してローカルフォルダに MD ファイルを直書きするだけのシンプルな構成でした。Obsidian プラグインすら噛んでいない、本当に自分用のやつ。

これを友人に見せたときに反応が良くて、「n8n で組むと半日かかるから、月 1,000 円で使える形にするならやる?」って聞いたら **即決で「やる」** と返ってきました。1,000 円で迷わず即決される需要があるなら、これは出していいやつだなと判断して着手しました。

Cloudflare Workers / Workers AI / D1 で組めば、変動費はほぼ 0 円・固定費は最初の開発工数だけ、と試算が立っていたのも大きかったです。

開発は AI による実装速度を活かして、2026 年 1 月の着手から **約 2〜3 ヶ月でストア公開** まで持っていきました。

### やってみてわかったこと

- Cloudflare スタックをプロダクションで回すとどんな感じか、の土地勘ができた
- 個人サービスで AI を組み込むなら、まず埋め込みモデルで解けないかを検討するのが圧倒的に良い。LLM のブレが運用負荷になる場面が結構ある
- 拡張機能 / Web / API / Obsidian / リアルタイム同期を横断する設計は、モノレポで型を共有しないと結構しんどい
- 1,000 円の課金意思を先に確認してから着手する動き方は、Swept での仮説検証の習慣がそのまま効いた。N=1 の判断だが、これがあったから「売れないかも」と悩み続けずに公開まで一気に走れた
