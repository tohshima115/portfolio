## TL;DR

3 人チームの社会起業プロジェクト「Swept」に約 2 年関わりました。前半 1 年は旅行をテーマにユーザーインタビューをひたすら回して撤退、後半 1 年は「愛着」にテーマを切り替えてロゴ / 名刺 / Web を一気通貫で作りました。最初は「デザイナーとして入ったつもり」だったのに、抜ける頃には「自分で設計したものを自分で実装してる人」になっていたのが、自分にとっての一番の変化です。方向性の違いで 2026 年 1 月に離脱しました。

| 項目 | 内容 |
|---|---|
| 期間 | 約 2 年 (2024 — 2026.01) / 完全ボランティア・週 3 日相当 |
| 自分の役割 | デザイン (Logo / Brand / Web) + フロントエンド実装 + UX リサーチ |
| チーム構成 | 3 人 (自分 + CEO 役 + マネージャー役)。事業企画とリサーチは 3 人で |
| 主な成果物 | ロゴ / ブランドアイデンティティ / 名刺 / ホームページ |
| Status | 自分は離脱。Web の運用は残メンバーで継続中 |

## どんな関わり方だったか

シェアハウスを一緒に始めたメンバーで、「すぐミーティングできるように」と物理的に集まった状態でやっていた 2 年でした。週 3 日くらいを Swept に、残りはバイトという生活で、完全ボランティア。離脱時に「これまで作った成果物の対価」という形で経費分だけ精算してもらった、というくらいの距離感です。

役割は一応「自分はデザインと実装」「CEO 役」「マネージャー役」と分かれてはいたんですが、事業企画とユーザーインタビューはほぼ 3 人で全部やっていました。対外プレゼンや企業コミュニティでの交流みたいな「外」の仕事は他のメンバーに任せていた感じです。

## Phase 1 — 旅行テーマで 1 年、撤退するまで

最初の 1 年は「旅行」をテーマにひたすら仮説検証をしていました。**通算で 100 人は超えるユーザーインタビュー** をやった気がします。

実は旅行の中でも、ターゲットとペインを 3 回くらい振り直しています。

**(a) 骨伝導イヤホン × 観光ガイド**

自分も他のメンバーも骨伝導イヤホンを使いがちで、「観光ガイドのあり方を変えられそう」というところから入りました。でもこれ、明らかにペイン発信じゃなくてシーズ発信のやつでした。インタビューしてもピンと来ないし、「お前らが欲しいか?」と言われたら欲しくもない。早めに撤退。

**(b) 外国人観光客の荷物問題**

次に「外国人観光客、荷物多すぎ問題」。これはペインとしてはあるんですが、具体的に解くプランがどうしても非現実的になる。アポなしで浅草や空港まで行って外国人観光客に突撃インタビューを繰り返したんですが (この時期に度胸はめちゃくちゃ付きました)、「異国のサービスはそもそも信用できない」という声が刺さって、ターゲットを国内旅行に切り替えました。

**(c) 国内旅行の荷物 → 実は「後片付け」が本丸**

ここでカスタマージャーニーマップを使って細かく分解していったら、**「用意」自体はそんなに大きなペインじゃなくて、「洗濯と後片付け」が本丸** だということが見えてきました。じゃあ後片付け特化のサービスにしようか、というところまで行ったんですが、実装するイメージを真面目に詰めていくと「これ IT というよりコインランドリー的なローカル事業になるな」となって。**コインランドリーでよくない?** という結論に静かに着地して、ここで旅行テーマ自体を畳みました。

### 撤退判断の温度感

決め手は CEO 役の人の「中の違和感」でした。データ的にも「絶対これ行けるじゃん」ではなく、ある程度ニーズはあるけど想定の範囲を出ない、というレベル。自分ごと化できていないテーマだった、というのが今振り返ってもデカいです。

撤退提案はマネージャー役の人が最初に切り出して、自分も合わせて、サクッと決まりました。揉めた記憶はなくて、「そうだよね」「薄々感じてた」みたいな空気でした。

### 沈黙とCJMの話

副産物として、リサーチの作法を現場で覚えられたのは大きかったです。

**沈黙は本当に効きます。** 「○○ですよね?」みたいな確認で引き出すよりバイアスがかからなくて、ただ黙ってるだけで相手が勝手に補足情報を入れてくれる。これは普段のコミュニケーションでも、後から Web 制作会社で新人研修をやったときも、ずっと使っている技術になりました。

カスタマージャーニーマップは、個人でやる分には正直そこまで言語化しなくていい気もしているんですが、**3 人チームで「何を解こうとしているか」を 1 個の絵に揃える** という意味で必要なツールだったと思っています。上で書いた「実は後片付けが本丸」も、CJM を真面目に書かなかったら出てこなかったやつです。

## Phase 2 — 愛着テーマでロゴ / 名刺 / Web を作る

旅行を畳んだあと、「じゃあ次どうする?」となったときに、自分から「愛着というテーマが人生のど真ん中にある」という話をして、CEO 役の人が「お前が納得するんやったらそれしかないんちゃう」と返してくれて決まったのが Phase 2 のテーマです。

ここから自分は **ロゴ / ブランドアイデンティティ / 名刺 / Web の設計と実装** をまるっと担当しました。

### ロゴ: 卵 + S + 抱擁を黄金比の円で

ロゴは「Sweep = 掃き清める」みたいな話ではなくて、**愛着** をど真ん中に置いて作りました。

モチーフは 3 つを重ねています:

- **卵**: 守るべきものであり、「コロンブスの卵」的な発想の源でもあり、「自分たちもまだ卵」これからどんどん変わっていく、という意味
- **S**: Swept の頭文字
- **抱擁**: 抱きしめる動き

形は **すべて黄金比で作った円の組み合わせだけで構成** していて、後から誰が見ても再現できるロゴになっています。

実は 1 年目のかなり初期にもロゴは一度作っていたんですが、SNS のアイコンに置いたときにグラデーションが強すぎたり書き込みすぎていたりで明らかに良くなかったので、Phase 2 でシンプルに作り直しました。正方形のアイコン枠にきれいに収まる、SNS 耐性のある形を意識した、というやつです。個人的にもかなり気に入っています。

ブランド設計の入り方としては、いつも **キーワード抽出 → マインドマップ** から始めるタイプです。これ無しでは進められない。Swept のときも「我々の今の状況」「扱うもの」と相性のいいモチーフを連想で広げていった結果、卵が出てきた、という順番でした。

### 名刺 / DTP

ロゴから派生して名刺の組版・入稿もやりました。Web 育ちなのでトンボとか CMYK は「あー、これか」と思いながらやった記憶はあるものの、特に詰まるところはなかったです。

### Web を Astro で作った話

実装は Astro + Tailwind。当時の選定理由を素直に書くと、

- Cloudflare に置きたかったので Next.js は相性的になし
- Vue 系を知らないので React 系で探す
- ただのコーポレートサイトに Next ほどの装備は要らない、極論 HTML/CSS で十分
- 「もっとシンプルでよくない?」という気持ちが当時の自分に強かった

で、Astro が一番きれいに収まりました。今もう一度同じ要件で選んでも、たぶん第 1 候補は Astro のままです。

## 離脱について

2 年やって、2026 年 1 月に「方向性の違い」で抜けました。これは綺麗事の表現ではなくて、本当にそれだけです。

具体的に言うと、

- 自分の中のスタートアップ像と、Swept が進んでいる方向にズレが出てきた (福祉・社会起業寄りに重心が移っていた)
- お金の話に向かわず、コミュニティ形成や YouTube みたいな「プロダクト周辺」の活動に重心が分散していた
- 同じ時期に始めた個人開発のプロダクトの方が、「自分が解決したいこと」に近かった
- 完全ボランティアで、自分のやりたいことを封じ込めてまで続ける対価が、自分の中で見つけられなくなった

ただ、これは始まった時点から **「数年後にいなくなってても、まあそうだよね」という温度感を 3 人で共有できていた** 話でもあって。「何年で抜ける」みたいな明確な話はしていないんですが、「ずっとここにいるとは限らないよね」「とはいえ 1〜2 年はやってるやろ」くらいの温度で、ふわっと握れていたんだと思います。結果的にちょうど 2 年目で実際にそうなった、という感じで、「ついにこの時が来たか」くらいで受け取ってもらえました。事前にすり合わせができていたのは、お互いにとって良かったと思っています。

抜けた今もシェアハウスは一緒に住んでいて、「どんな感じ?」「ここで困ってる」みたいな話は普通にしています。仲はめちゃくちゃいいです。

## ここで持って帰ったもの

- 「デザインまで」だった自分が「設計から実装まで」になった転換点 (Cloudflare をガッツリ触り始めて、出荷まで自分で完結できると気付いたあたりで、自分の名乗りが Designer から Product Engineer にじわっと変わりました)
- ロゴ → 名刺 → Web の世界観を一人で通す経験
- ユーザーインタビュー / 沈黙 / CJM / 撤退判断という仮説検証の基本動作を、手を動かして覚えた
- **「ボランティアで、自分が本当に解決したいこと以外はやらない」** という、次に持っていく判断軸
