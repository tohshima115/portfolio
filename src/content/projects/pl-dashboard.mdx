## TL;DR

先輩が日々参照していた**見にくいExcel製のPLシートを、Cloudflare D1 + OOUIのダッシュボード**にリプレイス。**Cloudflare Zero Trust** で Google Workspace アドレスを持つ社内メンバーのみアクセス可能に制御。**自発的に提案・実装**した社内ツール改善案件。

## Context

入社して間もない頃、先輩が毎月の数字をまとめるのに使っていた Excel 製の PL シートを見せてもらいました。**フィルタ・ピボット・色分けが入り組み、編集の度に整合性が崩れる**典型的な「成長したExcel」状態。本人は慣れで運用していたものの、新しく入った人間が読み解くのは難しく、引き継ぎコストの観点でも持続性に欠ける状態でした。

「これ、ダッシュボードにした方が早い」と感じて**自発的に作ることを提案**。経営学部出身でPL構造への理解があったことが、データモデルの設計を素早く進めるうえで効きました。

## Approach

### 1. UX Research — 何が辛いのかを言語化
- どの数字を見て、どの数字と比較しているか
- どの粒度で集計・突合する必要があるか
- 先輩が「ぱっと見」で異常に気付く感覚を何で代替するか

これをヒアリングしたうえで、**OOUI（Object-Oriented UI）** の発想で「数字 → 操作」ではなく「**勘定科目・期間というオブジェクトを起点に画面を組む**」設計に切り替えました。

### 2. データ移行 — Excel → D1
ExcelのシートをCSVに落とし、勘定科目テーブル / 取引テーブル / 期間サマリテーブルに分解。**Cloudflare D1 (SQLite)** にスキーマを切って Drizzle で型安全にアクセス。

### 3. アクセス制御 — Cloudflare Zero Trust
社内ツールであり、しかも金銭関連の数字を扱うため、**「URLを知っていれば誰でも入れる」状態は避けたい**。Cloudflare Zero Trust を被せ、**Google Workspace のドメインアドレスを持つアカウントのみ通過**するアクセスポリシーを設定。これにより VPN や別途の認証実装なしで安全な社内公開を実現。

### 4. UI — 機能美としてのダッシュボード
- 勘定科目を**オブジェクト**として一覧 → クリックで詳細にドリルダウン
- 月次・四半期・年次のビューを切替可能
- 異常値（前月比急変など）に**警告色**を割り当て、目視チェックを高速化
- Tailwind CSS で素早く構築、印刷物デザイン経験から「**情報の優先順位を視覚的にコントロール**」する感覚を活用

## Outcome

- 「Excelを開く → 必要な数字に辿り着くまでの操作回数」が**目視で大幅に減少**
- 社内の誰がアクセスしても**同じ数字を見られる**ようになり、引き継ぎコストが消えた
- 「先輩が困っていたから自発的に作る」という行動を起こせたこと自体が、本案件の最大の価値

> Cloudflare D1 / Zero Trust の組み合わせは、**社内ツールに最適なスタック**だと実感しました。データベースを別途立てる必要がなく、認証も IdP 連携で済むため、1人のエンジニアでも安全な業務ツールを短期間で構築できます。
