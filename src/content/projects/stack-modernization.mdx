## TL;DR

入社時点で「Photoshop でウェブサイトをデザイン → HTML + jQuery → 社内オンプレへ FTP 公開」が残っていた社内に、Figma + Astro + Cloudflare を経営陣と相談しながら段階的に入れていった。気づいたらフロントエンドの標準を作り直していた。

| 項目 | 内容 |
|---|---|
| Role | 課題発見 / 提案 / 経営陣との合意形成 / 技術選定 / 導入伴走 |
| Before | Photoshop でデザイン / HTML + CSS + jQuery / 社内オンプレ FTP |
| After | Figma でデザイン / Astro + React / Cloudflare Pages + Workers |
| Status | 社内標準フレームワークが Astro に置き換わり、他メンバーにも浸透 |

## Context

入社して間もなく、いくつかの「これは構造的に厳しい」運用が見えてきました。

- **Photoshop でウェブサイトのデザインを作っている**（小数点付きの px 値、グリッド未設計、デザインシステムなし）
- コーディングは **HTML + CSS + jQuery** が標準
- 公開は **社内オンプレサーバーへ FTP アップロード**

Web 制作を主業の一つに置く事務所としてはモダンな手元から大きく離れており、とくに Photoshop で Web をデザインする運用は、**デザイン → 実装の橋渡しコストが構造的に高くなる**原因になっていました。

ただ「全部いきなり変える」のは現場負担が重いため、**経営陣と相談しながら段階的に切り替えていく**方針で進めました。

## Approach

### 1. デザインツール: Photoshop → Figma へ
新規の Web 案件は **Figma で進める**ことを標準に切り替え。Adobe を完全排除はせず、**DTP（印刷物入稿）用途では Illustrator / Photoshop を残す** 棲み分けにしました。デザインの取り回し・レビュー速度・実装連携（CSS 出力・スペック共有）のすべてが改善。

### 2. フロント: jQuery → React / Astro へ
自分が担当する案件では最初から **jQuery を一切使わず React で実装**。徐々に他メンバーにも浸透し、複数人で扱える共通フレームワークが必要になった段階で、**Astro を社内標準**に置きました。

Astro を選んだ理由：

- **多ページアプリ（MPA）の作法**が、もともと静的 HTML を書いていた人たちにとって違和感が少なく、**移行コストが低い**
- **React / Vue / Vanilla JS のいずれでも書ける**柔軟さ ── 案件ごとに最適な選択がしやすい
- ページルーティング・API 配置の最低限の規約はあるので、自由度が高すぎて発散することもない
- 「フレームワークの縛りがゆるい / でも背骨はある」というバランスが、**まだ社内の技術標準が決まっていない事務所**にちょうど合っていた

### 3. 公開先: オンプレ FTP → Cloudflare へ（最も右往左往したフェーズ）

社内オンプレサーバは Node.js 環境ではなく、既存運用に影響を出さずに React / Astro を載せる経路を順に試しました。

| 試行 | 内容 | 結果 |
|---|---|---|
| ① | 静的書き出し → 既存 FTP | 動くが、SSR / API 連携が将来できない |
| ② | VPS に Node.js 環境を自前構築 | 構築・運用コストが見合わない |
| ③ | セルフホスト型 PaaS (Coolify 等) | 安定運用にもう一段の整備が必要で見送り |
| ④ | Vercel | 1 人なら最高だが、**複数人 Git 連携でシート課金が線形に膨らむ** 設計で社内利用に不向き |
| ⑤ | **Cloudflare Pages / Workers** | **人数課金なし・Astro サポート厚い・固定費が読める**ため採用 |

ちょうど **Cloudflare が Astro を取得** したタイミングで、社内のフレームワーク標準を Astro に揃える判断と整合 ── これは運の要素も大きかったですが、結果として導入の論拠が一段強くなりました。

## Outcome

- デザイン → 実装 → 公開までの一連の運用が、Web 業界の標準的なツールチェーンに揃った
- Figma / Astro / Cloudflare が他メンバーにも浸透して属人化が緩和
- バイトリーダーとして取りまとめを任されるようになった
- 新人が入っても Astro の作法と Figma → CSS の橋渡しで立ち上がりが速くなった

> 個人開発で触っていた Cloudflare / Astro がそのまま社内の選定材料として使えたのはよかった。個人でやっていたことが思わぬところで役に立つ。
