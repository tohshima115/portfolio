## TL;DR

週3時間かかっていた**金額関連の手入力作業を AI で自動化**し、**週20分に短縮（年間約130時間削減）**。金銭を扱う業務特有の**ヒューマンエラーリスクも同時に低減**。

| 項目 | 内容 |
|---|---|
| Role | 課題発見 / プロンプト設計 / 実装 / 運用 |
| Stack | TypeScript / LLM API / OCR / Google Apps Script / Spreadsheet |
| 効果 | **週3h → 週20分** / **年間 約130時間** 削減 / 入力ミス削減 |
| 公開範囲 | 社内運用ツール（クライアント情報のため非公開） |

## Context

デザイン事務所の業務には、見積書 / 請求書 / 経費精算など**「数字を別の場所に転記する」作業**が日常的に発生していました。

- 紙の領収書 → 経費表へ転記
- 取引先 PDF → 集計シートへ転記
- 銀行取引明細 → 仕訳表へ転記

これらが**週3時間**を消費しており、なにより**金銭を扱うため1桁のミスが致命的**。誰かが時間を使って「ミスらないように集中して写経する」状態は、**人間がやるべき仕事ではない**と判断しました。

## Approach

### 1. 課題の分解
転記作業を「**抽出**（どこから数字を取るか）」「**変換**（どの形式に揃えるか）」「**書き込み**（どのシートのどの行へ）」の3段階に分解。それぞれにふさわしい技術を割り当てる方針にしました。

### 2. 抽出 — OCR + LLM
画像 / PDF からの読み取りは OCR で初段の抽出を行い、**LLM で「この書類は何で、どの数字が金額本体か」を判断**させる構成。LLM をルールエンジンとして使うことで、**フォーマットが少し違うだけで壊れる脆さ**を回避。

### 3. 変換 — プロンプト設計でルール化
- 税込 / 税抜 の判定
- 通貨記号・カンマ区切りの正規化
- 日付フォーマットの統一

これらは**プロンプトに業務上のルールを埋め込んで**LLMに代行させ、コードはバリデーションのみに専念させました。

### 4. 書き込み — Apps Script + Spreadsheet
最終的な書き込み先は社内が日常的に使う Google Spreadsheet。**Apps Script で行を特定して挿入**し、人間は最終確認だけで済む状態に。

### 5. ヒューマンエラー対策
- **必ず人間の最終承認**を経てから本シートに反映
- 既存値と差分が大きい場合は**警告フラグを立てる**
- 元の証憑（画像）へのリンクを残し、**遡れる状態**を保証

## Outcome

| 指標 | Before | After |
|---|---|---|
| 所要時間 | 週 約3時間 | 週 約20分 |
| 年間換算 | 約 156時間 | 約 17時間 |
| 削減効果 | — | **約 130時間 / 年** |
| 入力ミス | 月数件 | ほぼゼロ（最終承認時の検出のみ） |

> 「人間にやらせるとミスが起きる作業は、AI にやらせて人間は最終承認に徹する」という分業の組み替えが、**業務改善の本質**だと再確認できた案件。経営学部時代に学んだ「**標準化・分業による生産性向上**」の発想が、AI 時代でもそのまま効くことを実感しました。
