## TL;DR

プロジェクト単位で語るほどではない**ロゴ・グラフィック・UIモック・印刷物**などの小〜中規模制作のアーカイブ。**DTP（入稿対応）まで一人で完結**できるため、Web に閉じない仕事を依頼可能。

| 項目 | 内容 |
|---|---|
| 領域 | Visual Identity / Logo / Graphic / Editorial / UI Mock / DTP |
| Tools | Illustrator / Photoshop / Figma |
| 立ち位置 | フロント・バックエンドの実装にも踏み込めるデザイナーとして、デザイン単発の依頼にも対応 |

## Approach

領域ごとに思考のフレームを切り替えながら、共通する**「ひとつの軸」**を持って制作しています。

- **Visual Identity**: 名前・由来・受け手の感情を起点に、ロゴ／カラー／タイポグラフィの3点セットで世界観を定義。
- **Graphic / Editorial**: グリッドと余白を使った情報設計から入り、最後に「つかみ」となるビジュアル表現を上に乗せる。
- **UI Mock**: 単発のスクリーンであっても、**その背後にある利用文脈と導線を必ず仮定してから描く**。
- **DTP**: 印刷入稿の制約（裁ち落とし / トンボ / 色再現）を踏まえた制作が可能。Web 専業のデザイナーには出せない品質を担保。

## Why this matters

> プロダクトエンジニアを志す上で、**デザインができることは「実装と同じレイヤーで意思決定できる」武器**です。

エンジニアリング側の判断（情報密度・レイアウト・色の優先順位）に対して、**「なぜこのデザインを選んだか」を自分で答えられる**ことが、プロダクトの完成度を最後の数%押し上げます。本サイトの「機能美としてのインダストリアルUI」も、この自前の判断軸の延長線上で組まれています。
