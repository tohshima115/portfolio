## TL;DR

仕事の文脈から外れた、**利益度外視で作る** ものの置き場です。儲ける気のないネタや、自分が面白いと思う偏愛、新しい技術の素振り。**「うまくいかなくても別にいい」** のテンションでやれる場所を、意識的に分けて持っています。

| 項目 | 内容 |
|---|---|
| ルール | 期間は短め / 小さく完成させて出す / 受けなくても気にしない |
| 立ち位置 | 仕事の制約 (期限・やり切り) から外れた、ネタ起点でやる場 |
| 出力イメージ | **ネタツイの代わりに Web アプリにしてみる**、くらいの距離感 |

## なんで仕事と分けてるか

利益度外視でやるところ、というのが一番デカいです。儲けようという意思がそこにない。

仕事と分けない方式にすると、何がしんどいかというと、**うまくいかなかった時にしょげそう** だな、というのが大きい。それなら最初から「うまくいかなくても別にいい」精神でやるものとして切り分けておいた方が、自分の中で健全に回せる。

あと、期間を短めに設定しているのも結構大事だと思っていて。仕事だと「いつまでに何々やる」「途中で投げ出さない」みたいな制約が当然入ってきますが、遊びは **途中で投げ出しても別にいい**。そのノリでできる場所として価値がある、という感覚です。

逆に、本気でやるなら最初から本気でやる、という切り分けはちゃんとしている方だと思っていて、**「遊びから本気に育ったケース」みたいなのはほぼない** です。やる時はもう「これは本気」とフラグを立てて始めている。下の復活の呪文も、最初から「絶対金にならないやつ」と分かった上で作ったやつです。

## 置いてあるもの

### 復活の呪文ジェネレーター

URL を「一見意味のないひらがな列」に可逆変換する Web ツール。ドラクエ 2 のふっかつのじゅもんの対応表をそのまま借りて、URL ⇄ 64 文字ひらがなの双方向変換を実装しています。

きっかけは、ふと「**自分の手で暗号化的なものを作れないかな**」と思った瞬間でした。日本で有名な古典的な暗号化と言えば、ドラクエの復活の呪文だよな、という連想で出てきたやつです。

実際の使いどころは正直マジでない。強いて挙げるなら、**保存しておきたいアダルト系サイトの URL を、誰かに見られても大丈夫な形式で残しておく** ような用途なら、ジョークとして使えるかもしれない、という温度感です。いわゆる本気の暗号化というよりは、ちょっと面白みがある変換器、というくらいの位置付けで作りました。

技術的なメモとしては:

- DQ2 の対応表は検索ですぐ出てきたので、そのまま流用
- 元から **64 文字に普通に収まる** ので、特に変えるところがなかった
- 複合化がちゃんと戻ることを担保したくて、双方向のテストは書いた (はず)

Twitter に貼ってバズるとかは **全然なかった** です。かすりもしなかった。でも、遊びで作ってるものなので、ユーザーが付かなくても OK というテンションで出していて、それは事前の想定通り。自分で使う場面もほぼないけどギリ使いそう、くらいで満足しています。

### チクチク心理診断 (制作中)

もうちょっと整えたら出すつもりで、現状は手元で動いているネタツール。

世の中の心理診断、**誰にでも当てはまることを言って気分良くさせるやつが多い** と思っていて。「あなたは芸術家タイプです」って言われたら、そりゃ誰でもちょっと嬉しくなる。**それがしょうもないな** と思ったところから始まりました。

そうじゃなくて、「理想を追い求めて気に病めない人ですね」みたいな **厳しい言葉だけで返してくる心理診断** があったら面白いんちゃう?という思いつき。**ぽかぽか言葉 ⇄ チクチク言葉を切り替えるボタン** を付けて、見比べながら「ほら、普段こんな感じで気分良くなっちゃってますよね」と見せられる作りにしようとしています。信憑性とかはない、ネタです。

気持ちとしては、**ネタツイの代わりに Web アプリにしてみる**、くらいの距離感です。Twitter で適当なことを言って終わるのを、形にして使えるものにして出してみる、というだけ。お金になることはなさそうだし、それで全然 OK です。

## 時間どこから出してるか

これは Hobby を続けるうえで地味に大事な話で、**週に 1〜2 日は「ちゃんとしたアウトプットをしなくていい日」を意識的に作っています**。

その日はインプットしてもいいし、新しい技術を試してもいいし、何ならボーッとしてるだけでもいい、という枠です。そういう日に何か作ったり、ネタを形にしたりすることが多い。

なんでこの日を設けてるかというと、**アイデアが出てくる瞬間は、ガーッと作業してる時じゃなくてちょっと休憩してる時** だな、というのが実感としてあるからです。気を抜けてる時、アウトプットしてない時にアイデアが振ってくることが多い。だったら週に 1 日くらいは、意識して何もアウトプットしない時間を確保しておかないと、そもそも次に作るものが湧いてこないよな、という運用です。

## やっててわかること

「Hobby で触った技術が仕事にバンバン活きてきます!」みたいな、絵に描いたような循環が起きてるかと言われると、**正直そんなにない** です。たまに「あ、これ前に遊びで触ったやつだ」くらいのことはあるけど、そこまで鮮やかに繋がってくるシーンは多くない。

ただ、何が良いかと言うと、**「行動する / コードを書く / Web に公開する / そのためにデザインを考える / 実装面で頭を回す」みたいな一連の動きを、遊びの中にそのまま組み込めている** こと自体が、自分にとってはデカいなと思っていて。

土日とか休みの日に、義務感ゼロで「ちょっとこのネタ形にしてみるか」のテンションでこれが回ってる、というのが結構いい状態だなと思いながらやっています。なんか作ってないと手が鈍る、みたいなのは普通にあるので、**遊びの体裁を借りて手を動かし続けてられる場所** として持っとくのが、たぶん一番の効用です。
