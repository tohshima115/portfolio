## TL;DR

仕事の文脈から外れた実験の置き場です。新しい技術スタックの土地勘を作るために何か作ったり、ただ「自分が面白いと思う偏愛」を形にしたものを置いています。**「小さく完成させて公開する」** のだけはルールにしています。

## 置いてあるもの

### 復活の呪文ジェネレーター

URL を「一見意味のないひらがな列」に可逆変換するだけの Web ツール。ドラクエの「ふっかつのじゅもん」をモチーフに、URL ⇄ 64 文字ひらがな（DQ2 の対応表をそのまま流用した Base64 亜種）の双方向変換を実装しています。

- 双方向変換のユニットテストはちゃんと書いて整合性を担保
- ひらがなだけの URL は、Twitter の DM に貼ったときに **面白い見た目** になる、というだけの遊び
- ジョークみたいなツールでも、設計と実装はちゃんとやる、の練習

## なんで Hobby を分けているか

仕事ではあんまり取れない振る舞いを、ここで意識的に練習している感じです。

- **小さく完成させる**: 機能を欲張らず、自分が一番楽しいと思う"芯"だけ作って公開する
- **未知の技術を持ち込む**: 仕事で採用するには早い新しいスタックや API を、まずここで触って土地勘を作る
- **ネタ / 偏愛を恥じない**: ターゲットが自分しかいないので、「面白い」を最優先で意思決定する

## やっててわかること

Hobby で触った技術は、なんだかんだいつか仕事のプロジェクトに循環してくるパターンが多いです。AIChatClip の Workers AI / Durable Objects の選定も、もとは Hobby で触った経験が判断の土台になっていました。遊んでいるのが、結局一番の素振りになっている気がします。
