## TL;DR

**音声入力 × AI** で、**カレンダー予定登録と LINE への定型登録業務を自動化**。同僚2人が「予定登録できていないことが頻繁に起きている」状況を見て、**自発的にチーム支援ツール**として制作した1人開発。

| 項目 | 内容 |
|---|---|
| Role | 課題発見 / 設計 / 実装 / チームへの導入 |
| Stack | Whisper (音声→文字) / LLM (構造化抽出) / Google Calendar API / LINE Messaging API |
| 動機 | 「同僚が予定登録できていない」という観察から始まった**チーム支援** |
| 公開範囲 | 社内運用 |

## Context

事務所内で**「予定の登録が間に合っていない」「LINE への報告漏れがある」**という状況が散見されました。本人たちが怠惰なのではなく、**「電話を切った直後・打ち合わせ直後」のような、PC を開いて入力する余裕がない瞬間**に発生する情報をすくえていなかったのが原因。

頼まれていない仕事ですが、**「ボトルネックが見えているのに放置するのは気持ち悪い」**と感じて、3日ほどでプロトタイプを作り、同僚2人に使ってもらいました。

## Approach

### 1. 観察ベースの課題定義
同僚に「困っていることは？」と聞くのではなく、**実際に起きている事象**を観察したうえで設計。
- 電話の直後にカレンダー入力する気力が残っていない
- LINE に書く言い回しが毎回ブレる
- 「あとでやる」と思っていると忘れる

→ **「PCを開かずに、声で吐き出すだけで終わる」**を体験のゴールに設定。

### 2. 入力UI — スマホで声を吐き出すだけ
スマホから音声入力するだけで動くフロントエンドを最小実装。
ボタン1つで録音 → サーバーへPOSTする構成にし、**操作回数を最小化**。

### 3. 音声 → 構造化
- **Whisper** で文字起こし
- **LLM で構造化抽出**（日時 / タイトル / 場所 / 参加者 / LINE 文面の要否）
- 曖昧な日時（「来週の水曜の午後」）も**現在時刻基準で確定**させる

### 4. 出力 — Calendar / LINE への分配
- 予定として登録すべきもの → **Google Calendar API**
- LINE への定型連絡が必要なもの → **LINE Messaging API**
- 両方必要なものは並列で実行

### 5. 失敗時の挙動
誤抽出は必ず起きる前提で、**「登録前に確認用メッセージが LINE に届く → 1タップで承認**」というUX。本人が画面を開かずに親指1本で済むことが、**継続利用の鍵**でした。

## Outcome

- 同僚2人の**予定登録漏れ・LINE 連絡漏れが解消**
- 自分自身の予定登録もこのフローに乗せて運用中
- **「頼まれていない仕事を自発的に作る」**という行動の証明として、面談時に語れるエピソード

> 業務改善は「困っている人に頼まれて作る」だけが正解ではない、と感じた案件。**観察で課題を定義し、一旦作ってしまって使ってもらうほうが速い**ケースが組織には多数存在します。経営学部出身で**事業全体の流れを観察する習慣**があったことが、課題発見の解像度に貢献していると思います。

クライアント・同僚の業務情報を含むため本ツール本体は非公開ですが、**音声→構造化→分配の設計**は AIChatClip の Webhook 連携と発想が共通しており、横展開可能な汎用パターンとして整備しています。
